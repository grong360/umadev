# 评测驱动交付（商业级必读）

> "在几个演示样例上看起来对"不是交付证据。商业级交付要让**评测成为方向盘**：用会**随线上失败不断生长的黄金集**衡量真实质量、不只看最终答案而是**评测整条轨迹**（选了什么工具、参数对不对、走了几步、花了多少、有没有违规）、用**校准过的裁判**判主观项、对关键检查要求 **pass^k**（连续 k 次都对）的可靠性而非碰巧一次过。
> 这份规范把评测从"跑个 demo"升级成可阻断的交付门，是 `agentic-delivery/01-standards/self-improving-memory-and-regression-sets`（回归集生长）在质量度量侧的延伸，与 `ai/agent-evaluation-benchmark`、`ai/02-playbooks/llm-evaluation-playbook`（产品侧评测体系）分工：本规范回答的是"凭什么相信这次交付是稳的，而不是恰好这次过了"。

## 1. 黄金集：由线上失败生长，而非一次性手搓

- **起点**：核心任务的代表性用例 + 已知边界与错误场景。
- **生长**：每一个逃到线上的失败、每一个被抓到的回归，都**回灌**成黄金集里的一条新用例——评测集随真实失败单调变强，覆盖你真正会踩的坑。
- **分层**：核心场景 / 边界场景 / 历史失败回归，按业务优先级分层维护。
- **与回归集衔接**：项目级回归集（见 `agentic-delivery/01-standards/self-improving-memory-and-regression-sets`）是黄金集中"必须永远对"的硬子集。

## 2. 轨迹评测：不只看最终答案

最终答案对、过程却荒唐（瞎调工具、绕一大圈、违反策略），在交付里同样是失败。要评**整条轨迹**：

| 轨迹维度 | 评测点 |
|---|---|
| 工具选择 | 是否选了正确的工具/动作，而非碰运气堆调用 |
| 参数有效性 | 工具调用的参数是否合法、符合契约、指向真实接口 |
| 步数 / 效率 | 是否用合理步数达成，有无无意义绕路与重复 |
| 成本 | token / 工具调用成本是否在预算内 |
| 策略合规 | 是否触发越权、越界、违规操作（安全/权限/数据） |
| 最终结果 | 产出是否满足验收标准 |

原则：**结果正确是必要不充分条件**；轨迹的工具选择、参数有效性、合规性同样进评测，否则会奖励"歪打正着"。

## 3. 校准裁判与裁判黄金集

主观判定（"这个产出合不合格""需求满没满足"）若交给一个未经校准的裁判，等于换了把不准的尺子：

- **裁判黄金集**：维护一组**已知正确标注**的样例（人工标定对/错），用来量裁判本身的准确率。
- **校准**：裁判必须先在裁判黄金集上达到既定一致率，才允许用它判真实产出；裁判准则变更要重新校准。
- **可解释**：裁判输出 pass/fail 要带证据（违反哪条标准），而非一句"感觉不行"，便于复核（呼应 `agentic-delivery/01-standards/verifier-critic-pattern`）。
- **防漂移**：定期用裁判黄金集复测，发现裁判漂移即重校准。

## 4. pass^k：关键检查要的是可靠，不是碰巧

单次通过证明"可能行"，不证明"稳定行"。对**关键路径/高风险**检查，用可靠性口径：

| 口径 | 含义 | 用在哪 |
|---|---|---|
| 单次通过 | 跑一次过 | 一般/低风险检查 |
| **pass^k** | **连续 k 次独立运行全部通过**才算过 | 关键路径、支付/权限/数据完整性、不可逆操作 |

- 关键检查要求 pass^k，把"偶尔过"挡在门外；k 随风险等级上调。
- 与之配套：flaky（时绿时红）用例视为不可靠，必须先稳定或隔离（见 `testing/01-standards/ci-test-gates-and-coverage`），不允许靠重试蒙混过 pass^k。

## 5. 评测作为交付门

- **门禁阈值**：关键场景通过率不得低于阈值；关键检查满足 pass^k；安全/合规违规为零方可放行。
- **回归触发**：实现、提示、工具、依赖任一变更都触发黄金集回归评测，趋势留痕。
- **四维趋势**：质量、可靠性、成本、合规随交付批次上报，退化超阈值阻断。

## 6. 接入交付流程

- **建集**：核心 + 边界 + 历史失败回灌，维护裁判黄金集。
- **评测**：每次关键变更跑黄金集；评轨迹不只评结果；主观项过校准裁判。
- **判定**：关键检查 pass^k、违规为零、阈值达标才放行。
- **生长**：线上失败与新回归回灌黄金集，评测集越用越强。

## 7. 反模式（出现即不合格）

1. **拿演示样例当交付证据**：几个 happy case 绿了就签字，不做标准化评测。
2. **黄金集一次手搓后不再生长**：线上失败不回灌，评测集与真实风险脱节。
3. **只评最终答案**：放过瞎调工具、违规操作、离谱成本的烂轨迹。
4. **用未校准裁判判主观项**：尺子本身不准，判定不可信。
5. **关键检查只看单次通过**：偶尔过被当成稳定，靠运气上线。
6. **靠重试刷过 pass^k**：flaky 用例不修，用重试掩盖不可靠。
7. **不跟踪成本与合规退化**：只盯正确率，安全/成本悄悄恶化。

## 8. 最低交付 checklist

- [ ] 黄金集覆盖核心 + 边界，并由线上失败与回归单调回灌生长。
- [ ] 评测覆盖整条轨迹：工具选择、参数有效性、步数、成本、合规，不只最终答案。
- [ ] 主观判定用经裁判黄金集校准的裁判，输出带证据、定期复测防漂移。
- [ ] 关键路径/高风险检查要求 pass^k（连续 k 次全过），k 随风险上调。
- [ ] flaky 用例先稳定或隔离，不允许靠重试蒙混过门。
- [ ] 实现/提示/工具/依赖任一变更触发黄金集回归评测，四维趋势留痕。
- [ ] 评测门阈值（通过率/pass^k/零违规）达标才放行，退化阻断。
